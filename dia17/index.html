Media Queries

¿Qué son?

Declaración en CSS que nos permite definir como se va a visualizar nuestra web

Sintaxis

Una MQ se compone de múltiples partes:

  • Keyword: @media // Es una palabra clave reservada
  • MediaType: un tipo de medida. (Usualmente "screen"/"print"/"all")
  • Operador/es: and/not/only
  • Breakpoints: condición a cumplirse (ejemplos: max-width, min-width...)
  • Reglas CSS: bloque de reglas

MediaType (Screen/Print)

- Print: destinado a material de vista de impresión
- Screen: visualización normal, en pantalla a color (utilizado por defecto)
- All: Todos los dispositivos
- Otros tipos que ya no se usan: tv, handheld, speech, braile...

Tips para utilizar MediaQueries

  1. Los MQ sobreescriben las reglas de CSS anteriores, PERO no poseen mayor especifidad que las otras reglas, por lo que debemos agregarlas al final del documento.
  2. Debemos utilizar los mismos selectores que queremos reemplazar/modificar
  3. Pensar bien los breakpoints
  4. Podemos utilizar una @media para cada selector, o utilizar una sola para todos los selectores (recomendada)

Mobile vs Tablet vs Desktop

Porcentaje de uso y medidas de pantalla:
- Mobile: 55% // 320px - 768px
- Tablet: 2% // 768px - 1024px
- Desktop: 43% // 1024px - 1920px o más
MOBILE FIRST. ¿Por qué? Es el dispositivo más utilizado. Es más "difícil" diseñar para mobile, porque la pantalla es más pequeña y tenemos menos espacio.

Uso de MQ en HTML

Podemos utilizar directamente MQ en el archivo HTML utilizando el atributo [media=""]. Antes que nada, debemos asegurarnos de que el head de nuestro HTML incluya la etiqueta meta de viewport
(meta name="viewport" content="width=device-width, initial-scale=1.0")

  • etiquetas de link:
    < link rel="stylesheet" href="./css/tablet.css" meta="screen and (min-width:768px)" >
  • etiqueta en style:
    'style media="stylesheet"
  • etiqueta de picture (recomendable)
